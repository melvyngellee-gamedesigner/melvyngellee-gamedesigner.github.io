My Story

My journey in the gaming world didn't begin in front of a screen, but around a table. Playing board games taught me to dissect mechanics before I ever touched a game engine: understanding the balance of rules, the importance of player chemistry, and the satisfaction of a well-oiled system. This curiosity naturally extended to video games as I grew up, where I learned to observe level architecture, environmental storytelling, and, of course, the mechanics themselves.

My approach to Game Design is the result of a balance between precision and emotion.

Passionate about the works of Tolkien and the density of universes like those of Cyberpunk 2077 or League of Legends, I see Narrative Design as complex architecture where even the smallest metaphorical detail reinforces immersion.

This pursuit of precision and emotion is also reflected in my inspirations, such as Frédéric Chopin: from my Franco-Polish roots, I inherited a sensitivity for virtuoso complexity placed at the service of emotion. This translates even more into my approach to UX and Game Feel: an invisible technical complexity that allows the player to feel a pure and fluid emotion.

On the court, my experience with volleyball gave me a sharp understanding of collective dynamics and the transmission of shared emotions. Whether through the mechanical depth of Rocket League or the strategic demands of roguelikes, I strive to seek experiences where system precision meets the satisfaction of pushing one's limits.

My goal is simple: to design rich worlds and deep mechanics that, once in hand, transform every session into an unforgettable moment of synergy.

01. THE PITCH

Ungor Bok is a platform game with one major constraint: the entire game is played with a single button.

The player controls a dwarf explorer in a legendary cave and must overcome platforming obstacles using only a grappling hook. Timing and precision are key.

02. MY RESPONSIBILITIES (SOLO DEV)

As the sole developer, I managed the entire production pipeline:

  • Level Design: Creating "Puzzle rooms" where understanding the momentum is more important than reflexes.
  • Game Feel & 3C: Iterating on the grappling hook physics to make it satisfying (Juice, Screen Shake, Particles).
  • Programming: Implementing the state machine for the character and the physics calculations.

03. TECHNICAL CHALLENGE: THE FEEDBACK

The main difficulty was accessibility. The player needed to intuitively understand the grappling angle without complex UI.

Solution: I implemented a rotating arrow around the player. It acts as a diegetic crosshair, helping the player time their input perfectly.

Code snippet used to create the grappling hook logic (Physics & Visuals).

Tutorial_GrapplingGun.cs C# 

using System.Collections;
using System.Collections.Generic;
using UnityEngine;

public class Tutorial_GrapplingGun : MonoBehaviour
{
    [Header("Scripts Ref:")]
    public Tutorial_GrapplingRope grappleRope;

    [Header("Layers Settings:")]
    [SerializeField] private bool grappleToAll = false;
    [SerializeField] private int grappableLayerNumber = 9;

    [Header("Main Camera:")]
    public Camera m_camera;

    [Header("Transform Ref:")]
    public Transform gunHolder;
    public Transform gunPivot;
    public Transform firePoint;
    public Transform grappleIndicator; // La flèche qui tourne autour du joueur

    [Header("Physics Ref:")]
    public SpringJoint2D m_springJoint2D;
    public Rigidbody2D m_rigidbody;

    [Header("Rotation:")]
    [SerializeField] private bool rotateOverTime = true;
    [Range(0, 60)] [SerializeField] private float rotationSpeed = 4;
    [SerializeField] private float indicatorRotationSpeed = 100; 
    [SerializeField] private float indicatorRadius = 1.5f;

    [Header("Distance:")]
    [SerializeField] private bool hasMaxDistance = false;
    [SerializeField] public float maxDistnace = 20;

    private void Update()
    {
        RotateIndicator();
        UpdateIndicatorColor();

        if (Input.GetKeyDown(KeyCode.Space))
        {
            SetGrapplePoint();
        }
        else if (Input.GetKey(KeyCode.Space))
        {
            if (grappleRope.enabled)
            {
                RotateGun(grapplePoint, false);
            }
        }
        else if (Input.GetKeyUp(KeyCode.Space))
        {
            ResetGrapple();
        }
    }

    void RotateIndicator()
    {
        indicatorAngle += indicatorRotationSpeed * Time.deltaTime;
        float x = Mathf.Cos(indicatorAngle * Mathf.Deg2Rad) * indicatorRadius;
        float y = Mathf.Sin(indicatorAngle * Mathf.Deg2Rad) * indicatorRadius;
        grappleIndicator.position = gunPivot.position + new Vector3(x, y, 0);
        grappleIndicator.up = (grappleIndicator.position - gunPivot.position).normalized;
    }

    void UpdateIndicatorColor()
    {
        Vector2 direction = (grappleIndicator.up).normalized;
        RaycastHit2D hit = Physics2D.Raycast(firePoint.position, direction, maxDistnace, 1 << grappableLayerNumber);

        if (indicatorSprite != null)
        {
             indicatorSprite.color = (hit.collider != null && (hit.collider.gameObject.layer == grappableLayerNumber || grappleToAll)) ? colorGrapplable : colorNotGrapplable;
        }
    }

    void SetGrapplePoint()
    {
        Vector2 direction = grappleIndicator.up.normalized;
        RaycastHit2D hit = Physics2D.Raycast(firePoint.position, direction, maxDistnace, 1 << grappableLayerNumber);

        if (hit.collider == null)
        {
            ResetGrapple();
            return;
        }

        grapplePoint = hit.point;
        grappleDistanceVector = grapplePoint - (Vector2)gunPivot.position;
        grappleRope.enabled = true;
    }

    public void Grapple()
    {
        m_springJoint2D.autoConfigureDistance = false;

        if (!launchToPoint)
        {
            m_springJoint2D.connectedAnchor = grapplePoint;
            m_springJoint2D.enabled = true;
        }
    }
}
